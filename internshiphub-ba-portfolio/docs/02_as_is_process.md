# 02. AS-IS Process Analysis

## 1. Purpose

This document describes the current internship recruitment process before implementing the InternshipHub system.

The goal of AS-IS analysis is to understand the current workflow, identify manual steps, find pain points, and discover improvement opportunities.

## 2. Current Process Summary

Currently, the internship recruitment process is managed through different tools such as email, spreadsheets, chat applications, and manual notes.

This process can work when the number of candidates is small. However, when the number of applications increases, HR may find it difficult to track candidate status, schedule interviews, collect feedback, and send timely updates.

## 3. AS-IS Process Steps

| Step | Actor | Current Activity | Tool Used |
|---|---|---|---|
| 1 | Candidate | Sends CV or fills application form | Email / Google Form |
| 2 | HR Recruiter | Downloads candidate CV | Email / Google Drive |
| 3 | HR Recruiter | Adds candidate information to tracking file | Excel / Google Sheet |
| 4 | HR Recruiter | Screens candidate profile manually | CV / Spreadsheet |
| 5 | HR Recruiter | Contacts candidate for next step | Email / Zalo / Teams |
| 6 | HR Recruiter | Schedules interview with interviewer | Chat / Calendar / Email |
| 7 | Interviewer | Reviews candidate CV before interview | Email / Shared folder |
| 8 | Interviewer | Conducts interview | Online meeting / Offline |
| 9 | Interviewer | Sends feedback to HR | Chat / Email / Spreadsheet |
| 10 | HR Recruiter | Updates candidate status manually | Spreadsheet |
| 11 | Hiring Manager | Reviews selected candidates | Spreadsheet / HR summary |
| 12 | HR Recruiter | Sends final result to candidate | Email / Chat |

## 4. Pain Points

| Pain Point ID | Pain Point | Impact |
|---|---|---|
| PP-01 | Candidate information is stored in many places. | HR may lose information or spend more time searching. |
| PP-02 | Candidate status is updated manually. | Status may become outdated or incorrect. |
| PP-03 | Interview scheduling is done through chat or email. | Time conflict or missing information may happen. |
| PP-04 | Interview feedback is not standardized. | HR and hiring manager may find it hard to compare candidates. |
| PP-05 | Candidate does not know the current application status. | Candidate experience becomes unclear and stressful. |
| PP-06 | Hiring manager cannot see the recruitment pipeline clearly. | Final hiring decision may be slower. |
| PP-07 | There is no clear status history. | It is hard to audit who changed what and when. |

## 5. Root Causes

| Root Cause ID | Root Cause |
|---|---|
| RC-01 | There is no centralized system for recruitment management. |
| RC-02 | HR depends heavily on spreadsheets and manual updates. |
| RC-03 | Candidate communication is not automated. |
| RC-04 | Interview feedback does not follow a fixed format. |
| RC-05 | Recruitment data is not connected across stakeholders. |

## 6. Improvement Opportunities

| Opportunity ID | Improvement Opportunity |
|---|---|
| OP-01 | Create a centralized candidate database. |
| OP-02 | Allow HR to manage candidate status in the system. |
| OP-03 | Add interview scheduling and interviewer assignment. |
| OP-04 | Create a structured feedback form for interviewers. |
| OP-05 | Allow candidates to view their application status. |
| OP-06 | Provide a recruitment pipeline dashboard for hiring managers. |
| OP-07 | Store status history for audit and tracking. |

## 7. BA Notes

The AS-IS process shows that the main problem is not only one missing feature. The real problem is that the recruitment workflow is fragmented across many tools.

Therefore, the future system should focus on centralization, transparency, standardization, and tracking.
